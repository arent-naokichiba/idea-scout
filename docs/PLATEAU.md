# PLATEAU ツール - 国土交通省 Project PLATEAU データアクセス & 3Dビューア

国土交通省が主導する [Project PLATEAU（プラトー）](https://www.mlit.go.jp/plateau/) の
データカタログAPIにアクセスするツール群。

| ツール | 内容 |
|---|---|
| `plateau_viewer.py` | **3D都市モデルビューア**（ブラウザで3D表示・検索・属性参照） |
| `plateau_cli.py` | コマンドラインでの検索・ダウンロード |

## PLATEAU Viewer（3Dビューア）

```bash
python plateau_viewer.py            # サーバ起動 + ブラウザが自動で開く
python plateau_viewer.py --port 8080 --no-browser
```

Windows は `run_viewer.bat`、macOS / Linux は `./run_viewer.sh` でも起動できる。
追加のPythonパッケージは不要（標準ライブラリのみ）。ブラウザ（Chrome / Edge / Safari / Firefox）が
CesiumJSで3D Tilesをストリーミング描画するため、Windows / Mac どちらでも同じように動作する。

### スマートフォン（iPhone / Android）から使う

```bash
python plateau_viewer.py --lan
```

`--lan` を付けて起動すると、コンソールに `http://192.168.x.x:8765/` のような
アドレスが表示される。**同じWi-Fiに接続した**スマホのブラウザ（Safari / Chrome）で
そのアドレスを開けば、タッチ操作対応のモバイルUI（ドロワー式メニュー）で利用できる。

- 初回はPC側でOSのファイアウォール許可のダイアログが出る場合がある（許可する）
- スマホでは品質スライダーを右寄り（高速）にし、建物はLOD1を選ぶと快適
- ピンチでズーム、2本指ドラッグで視点の傾き変更（Cesium標準のタッチ操作）

### 主な機能

- **3D表示（高速ストリーミング）** — 3D TilesのLOD制御により視点に応じて必要なタイルだけを読み込む。
  ヘッダーの「品質」スライダーで描画品質と速度のバランスを調整可能
- **都市計画情報の表示（MVT）** — 用途地域・防火地域・高度地区などの都市計画決定情報、
  土地利用、土砂災害警戒区域等のベクタタイルを描画。用途地域は都市計画図の標準配色で
  色分けし、凡例を自動生成。クリックで告示番号・建蔽率・容積率などの属性を参照可能
- **クイックスタート** — 東京・大阪・名古屋・札幌・福岡の建築物モデルをワンクリック表示。
  「用途地域を重ねる（都市計画）」「洪水浸水想定を重ねる（防災）」オプション付き
- **データセット検索** — 都道府県 / 市区町村 / 種別（建築物・洪水浸水想定区域など24種）で絞り込み
- **都道府県全域・年度別データ（経年比較）** — 検索対象を「都道府県全域（年度別）」に
  切り替えると、結合タイルセット（過去年度含む）を検索できる。同じ種別の年度違いを
  追加して左右比較モードで並べれば**都市の経年変化を比較**できる。
  東京都は建築物LOD3/LOD4（屋内含む詳細モデル）・地下街・道路LOD3もここから利用可能
- **全国ハザードマップ** — 重ねるハザードマップ（ハザードマップポータルサイト）の
  洪水・津波・高潮・土砂災害（土石流/急傾斜地/地すべり）をワンクリックでオーバーレイ。
  PLATEAU未整備の自治体もカバー
- **レイヤー管理** — 複数データセットの重ね合わせ、表示/非表示、透明度、ズーム、凡例
- **色分け表示** — 建物の高さによる色分け、単色（白模型風）表示
- **属性フィルタ** — 建物高さの範囲指定で絞り込み表示（例: 60m以上の建物のみ）
- **属性参照** — 建物・区域クリックで属性（階数・用途・建蔽率・容積率など）をパネル表示、JSONコピー
- **地形表示** — 地理院標高タイル（DEM10B相当）による地形の起伏。山間部の都市や
  浸水想定と地形の関係を立体的に確認できる
- **左右比較モード** — レイヤーごとに「左側のみ / 右側のみ / 両側」を指定して分割表示。
  中央のバーをドラッグして比較（例: 洪水の計画規模 vs 想定最大規模、LOD1 vs LOD2）
- **建物統計** — 現在のビューに読み込まれている建物を集計し、建物数・平均/最高高さ・
  高さ分布（地図の色分けと同じ配色）・用途構成をパネル表示
- **住所検索** — 国土地理院ジオコーダで地名・住所から移動
- **距離・面積計測** — クリックで測点を追加して距離/面積を計測（ダブルクリックで確定）
- **歩行者視点** — クリック地点に立って街並みを確認（W/A/S/D移動・←/→旋回・Q/E高さ）
- **断面クリップ** — 2点クリックで指定した鉛直面で3Dモデルを切断（Rキーで反転）。
  高層部の構成や高さ関係の確認に
- **日照シミュレーション** — 影の表示 + 時刻スライダー + 1日の影の動きのアニメーション再生。
  日付は今日 / 春分 / 夏至 / 冬至（日影規制の検討基準日）から選択
- **属性エクスポート** — 建物統計の集計対象をCSV（Excel対応・BOM付き）または
  GeoJSON（建物中心点 + 全属性、QGIS等で利用可）で保存
- **GeoJSONインポート** — 敷地境界・計画データ等のGeoJSONをファイル選択または
  マップへのドラッグ&ドロップで読み込み、地表・建物にドレープ表示。クリックで属性参照
- **BIMモデル配置（glTF / GLB）** — 自社の計画モデルをドロップ→配置地点をクリックで
  実際の街並みに設置。向き・高さ・スケールをレイヤーパネルで調整、再配置も可能
  （Revit / IFC は IfcConvert 等で glb に変換して利用）
- **クレーン配置シミュレーション** — 設置地点をクリックし、ブーム長・起伏角から
  作業半径とフック高を3D表示。「干渉チェック」で全方位にブームを振り、
  周辺建物と干渉する方位を赤い円弧で表示（施工計画・揚重計画の一次検討）。
  機種プリセット内蔵（ラフター25t〜オールテレーン350t）
- **車両走行パス検討** — 搬入ルートをクリックで作図すると、車種（2t〜大型トラック・
  ミキサー車・セミトレーラー・ポンプ車）の最小回転半径で各コーナーを旋回できるかを
  判定（可=緑 / 要切り返し=赤の旋回弧）。車幅+余裕の走行帯、進行方向の矢印
  （一方通行・運行方向の表現）、車両の簡易3Dモデルも表示。車幅・回転半径はカスタム入力可
- **ヤード作図** — 資材置き場・仮設事務所・車両待機・立入禁止区域・仮囲いの範囲を
  ポリゴン作図（種別ごとの色・高さで立体表示、面積自動計算、名称編集）。
  全ヤードをGeoJSON保存でき、インポートで再利用可能
- **近隣調査リスト** — 現場位置をクリック→半径を指定すると、範囲内の建物を
  距離順に抽出（用途・階数・高さ付き）。対象建物をハイライトし、全属性をCSV出力
  （近隣家屋調査の対象洗い出し）
- **視線・見通し解析** — 視点と対象の2点をクリックすると、建物による遮蔽を判定して
  見通し可（緑）/ 遮蔽区間（赤）と遮蔽地点までの距離を表示
- **HTMLレポート生成** — 現在の表示画面・レイヤー一覧・建物統計を1枚の
  自己完結HTMLとして保存（印刷・共有用）
- **工程管理** — 画面下部のタイムラインで工程（名称・開始/終了日・進捗率）を管理し、
  各工程にレイヤー（ヤード・クレーン・車両パス・BIMモデル等）を紐づけると、
  日付スライダー・再生に連動して**その時点の現場配置が再現**される
  （基礎工事期→躯体工事期のステップ切替、施工計画のプレゼンに）。
  localStorageに自動保存され単体で完結。外部連携は下記「工程データ連携API」参照
- **出来高表示（4D）** — 工程にリンクしたBIMモデルは、日付の経過に応じて
  **下から立ち上がる**（クリッピングによる出来高表現）。モデルの「全高(m)」で調整
- **4D動画書き出し** — 工程パネルの「🎬 動画」で工程アニメーションの再生を
  そのままWebM動画として保存（会議資料・施主説明用）
- **工程レポート** — HTMLレポートに工程表（ガントバー・表示日付カーソル付き）を自動掲載

### 工程データ連携API（外部の工程管理システムとの接続）

工程データは次のJSONスキーマで入出力する:

```json
{
  "version": 1,
  "project": "現場名",
  "tasks": [
    { "id": "t1", "name": "仮設工事", "start": "2026-07-01", "end": "2026-07-20",
      "progress": 40, "layers": ["zone-1", "crane-1"] }
  ]
}
```

連携方法は3系統:

| 方式 | 使い方 |
|---|---|
| **ファイル** | 工程パネルの「保存/読込」でJSONを入出力（どのシステムとも手動連携可） |
| **REST** | ローカルサーバの `GET/POST http://localhost:8765/api/schedule`（CORS開放済み）。外部の工程管理システムからPOSTでプッシュ→ビューアの「連携→サーバ読込」で反映。「連携→外部URL」に相手システムのAPIを指定して直接取得（60秒ごとの自動更新可） |
| **postMessage** | ビューアをiframeで自社システムに埋め込み、`iframe.contentWindow.postMessage({type: "plateau:setSchedule", schedule}, "*")` で工程を注入。`plateau:setDate`（日付カーソル移動）/ `plateau:getSchedule`（現在データの取得）/ `plateau:play`（再生）に対応 |
- **ベースマップ切替** — 航空写真 / 標準地図 / 淡色地図 / ダーク /
  **年代別空中写真（1961〜69年・1974〜78年）**。現在の3D建物と昔の街並みの比較に
- **ブックマーク** — 視点の保存・呼び出し
- **共有リンク** — 表示状態（カメラ・レイヤー・スタイル）をURLとしてコピー
- **セッション復元** — 前回の表示状態を自動保存・復元
- **スクリーンショット** — 表示中の3DビューをPNG保存
- **FPS表示 / 2D・3D切替**

### 表示の仕組み

```
plateau_viewer.py（ローカルHTTPサーバ）
  ├─ /              → viewer/ の静的ファイル（CesiumJSアプリ）
  ├─ /api/*         → データカタログAPIのプロキシ（ローカルキャッシュ利用）
  └─ /api/geocode   → 国土地理院 住所検索APIのプロキシ
ブラウザ → PLATEAU CDN（assets.cms.plateau.reearth.io）から3D Tiles / MVTを直接ストリーミング
```

- 3D Tiles（建築物・洪水浸水想定など）はCesiumが直接ストリーミング描画
- MVT（都市計画決定情報・土地利用など）は自前の描画エンジン（`viewer/mvt-layer.js`）で
  ベクタタイルをデコードしてイメージレイヤーとして描画。デコーダは
  pbf + @mapbox/vector-tile を `viewer/vendor/mvt.js` にバンドル済み（約10KB）
- MVTはズームレベル10〜16でデータが提供される（それより深いズームは親タイルを拡大描画）
- 地形は地理院PNG標高タイル（dem_png, z1〜14）を自前のTerrainProvider
  （`viewer/gsi-terrain.js`）でデコードして生成。海上などデータのないタイルは平坦扱い

※ タイル本体はPLATEAUのCDNから直接取得するため、インターネット接続が必要。

## PLATEAU CLI（コマンドライン）

- API: `https://api.plateauview.mlit.go.jp/datacatalog/plateau-datasets`（認証不要）
- カタログ（約9MB）は `data/plateau_catalog.json` に24時間キャッシュされる
- `--refresh` でキャッシュを無視して再取得できる（例: `python plateau_cli.py --refresh cities`）

### 自治体一覧（CityGML公開元）

```bash
python plateau_cli.py cities              # 全自治体（約300件）
python plateau_cli.py cities -q 東京      # 名前・団体コードで絞り込み
```

### データセット種別の一覧

```bash
python plateau_cli.py types
# 建築物モデル(bldg)、洪水浸水想定区域モデル(fld)、交通（道路）モデル(tran) など
```

### データセット検索

```bash
python plateau_cli.py search --city 千代田 -t bldg        # 千代田区の建築物モデル
python plateau_cli.py search --pref 北海道 -t 洪水 -n 50   # 種別は日本語名でも可
python plateau_cli.py search -q 渋谷 -f "3D Tiles"        # 形式で絞り込み
```

### 詳細表示・ダウンロード

```bash
python plateau_cli.py show 13101_bldg_lod2                # データセット詳細（URL等）
python plateau_cli.py download 13101_bldg_lod2            # データセットをダウンロード
python plateau_cli.py download 13101 --citygml -o data/plateau  # CityGML一式（zip）
```

- 保存先はデフォルトで `data/plateau/`（`-o` で変更可）
- 500MB超のファイルは確認プロンプトが出る（`-y` でスキップ）
- 形式が「3D Tiles」のデータセットはエントリポイント（`tileset.json`）のみ取得する。
  タイル本体はCesium等のビューアに `tileset.json` のURLを渡して利用するのが基本
- CityGML一式は自治体によっては数GBになるので注意（`cities` コマンドでサイズ確認可）

## Pythonから使う

```python
from idea_scout.plateau import PlateauClient

client = PlateauClient()
datasets = client.search_datasets(city="千代田", dataset_type="bldg")
for d in datasets:
    print(d["id"], d["name"], d["url"])
```

`PlateauClient` の主なメソッド:

| メソッド | 内容 |
|---|---|
| `list_cities(query)` | CityGML公開自治体の一覧 |
| `search_datasets(query, pref, city, dataset_type, data_format, limit)` | データセット検索 |
| `get_dataset(dataset_id)` | ID指定で1件取得 |
| `get_citygml(city_code)` | 団体コードでCityGML情報を取得 |
| `list_types()` | 種別一覧（件数付き） |
| `download(url, output_dir)` | ストリーミングダウンロード |
