## 注目の開発ネタ候補

### ★★★★ [garrytan/gstack](https://github.com/garrytan/gstack)
- ソース: GitHub Trending / AI
- 要約: Garry Tanが公開したClaude Code用の23ツールセット。CEO・デザイナー・エンジニアマネージャー・リリースマネージャー・ドキュメントエンジニア・QAなど複数ロールをAIに担わせる設定集。
- 開発ネタ: Revitアドイン開発やIFCツール開発にも同様のマルチロールAIエージェント構成を適用できる。BIM仕様レビュー担当・コーディング担当・テスト担当を分担させるClaude Code構成を整備すると、アドイン開発の効率が大幅に上がりそう。

### ★★★★ [DeusData/codebase-memory-mcp](https://github.com/DeusData/codebase-memory-mcp)
- ソース: GitHub Trending / DevTools
- 要約: コードベースを永続的な知識グラフにインデックス化する高性能MCP。158言語対応、ミリ秒以下のクエリ、トークン99%削減。単一バイナリでゼロ依存。
- 開発ネタ: RevitのAPIやIFCスキーマは巨大で複雑なため、このMCPでコードベースを知識グラフ化しておくとAIアシスト開発が劇的に改善できる。BIM SDKのラッパー開発やアドインの大規模リファクタに活用可能。

### ★★★ [Moebius: 0.2B image inpainting model with 10B-level performance](https://hustvl.github.io/Moebius/)

- 開発ネタ: 建築パースや平面図の欠損部分補完、BIMビューアでのレンダリング補助に活用できる。軽量モデルなのでRevitアドイン内に組み込むことも現実的。

### ★★★ [Show HN: Oak – Git alternative designed for agents](https://oak.space/oak/oak)

- 要約: AIエージェントのために設計されたGitの代替バージョン管理システム。エージェントが複数の変更を並列管理しやすい構造。
- 開発ネタ: BIMワークフローの自動化にAIエージェントを使う際、複数の設計変更案を並列管理するのに適している。IFCファイルの差分管理にも応用できるかもしれない。

### ★★★ [bytedance/deer-flow](https://github.com/bytedance/deer-flow)
- ソース: GitHub Trending / AI
- 要約: ByteDanceのオープンソース長時間タスク対応SuperAgentフレームワーク。サンドボックス・メモリ・ツール・サブエージェントを組み合わせ、数分から数時間かかるタスクを処理。
- 開発ネタ: IFCモデルの解析→問題検出→修正提案→レポート生成といった長時間BIM処理パイプラインをこのフレームワークで構築できる。複雑な建築法規チェック自動化にも応用可能。

### ★★★ [lyogavin/airllm](https://github.com/lyogavin/airllm)
- ソース: GitHub Trending / AI
- 要約: 4GB GPUのみで70B LLMの推論を可能にするライブラリ。モデルを分割してGPUメモリを効率利用する技術。
- 開発ネタ: 設計事務所のローカルマシンでも大型LLMを動かせるようになる。クラウド依存せずRevitアドイン内でローカルAI推論を行う実装が現実的になる。

### ★★★ [Stirling-Tools/Stirling-PDF](https://github.com/Stirling-Tools/Stirling-PDF)

- 開発ネタ: BIM/CADプロジェクトではPDF図面・仕様書の処理が多い。このツールを組み込んでIFC→PDF変換レポートの自動生成・結合・注釈付与ツールを作れる。
