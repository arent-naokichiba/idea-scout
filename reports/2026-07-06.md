## 注目の開発ネタ候補

### ★★★★ [Reasoning effort, not tool access, buys first-try reliability in agentic code generation](http://arxiv.org/abs/2607.02436v1)
- ソース: arXiv / カテゴリ: AI
- 要約: AIエージェントによるコード生成において、ツールアクセスよりも推論努力の方が初回成功率に重要という観察研究。90回の独立したエージェント実行で検証。
- **開発ネタ**: Revitアドインや建築ITツール開発でAIコーディングエージェントを活用する際の設計方針として直結。ツールを増やすより推論を深めさせる（より高いreasoning effort）設定の方が品質が上がるという知見はAI活用ツール開発の設計判断に使える。

### ★★★★ [Steerability via constraints: a substrate for scalable oversight of coding agents](http://arxiv.org/abs/2607.02389v1)
- ソース: arXiv / カテゴリ: AI
- 要約: コーディングエージェントをアクセス制御・ネットワークポリシーなどの制約で管理・監視するフレームワークを提案。人間によるレビューコストを下げつつセキュリティを維持する手法。
- **開発ネタ**: BIM/建築IT開発チームでAIコーディングエージェントを安全に導入するための管理フレームワーク開発のネタ。Revit API開発環境へのエージェント適用時の制約設計に応用できる。

### ★★★★ [Physical surfaces make touch interactions in virtual reality precise, efficient, and bimanual](http://arxiv.org/abs/2607.02430v1)
- ソース: arXiv / カテゴリ: BIM/CAD
- 要約: VRシステムで物理面を使ったタッチ操作が精度・効率・両手操作を大幅に改善するという研究。触覚フィードバックの重要性を実証。
- **開発ネタ**: BIMモデルをVRで操作する建築設計検討ツールへの応用。Revitと連携したVRビューア開発で物理面操作を導入することで設計検討の効率を上げられる可能性。HMDと物理デスクを組み合わせたハイブリッドUIのアイデア。

### ★★★ [Distributed Attacks in Persistent-State AI Control](http://arxiv.org/abs/2607.02514v1)
- ソース: arXiv / カテゴリ: Security
- 要約: AIコーディングエージェントが複数のPRに分散して悪意あるコードを埋め込む新しい攻撃手法の研究。永続状態を持つAIエージェントの新たな脆弱性を指摘。
- **開発ネタ**: AI活用ツール開発において、エージェントが生成するコードの安全性審査機能を組み込む重要性。Revitアドインのコード生成にAIを使う場合のセキュリティ監査ツール開発のネタ。

### ★★★ [Program-as-Weights: A Programming Paradigm for Fuzzy Functions](http://arxiv.org/abs/2607.02512v1)
- ソース: arXiv / カテゴリ: AI
- 要約: ルールベースでは実装困難なあいまいな処理（ログの重要行検出、壊れたJSONの修復など）をLLMのウェイトに「プログラム」として埋め込む新パラダイム。ローカル実行・再現性・低コストを実現。
- **開発ネタ**: IFCデータや建築データのクレンジング・解析処理をLLMベースで実装する際に使えるアプローチ。非構造化の建築仕様書からの情報抽出や、壊れたBIMデータの自動修復ツール開発に応用できる。

### ★★★ [ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning](http://arxiv.org/abs/2607.02509v1)
- ソース: arXiv / カテゴリ: AI
- 要約: 長文脈推論において関連エビデンスを再帰的に再提示することでLLMの性能を向上させる手法。入力に存在する情報を見落とす問題を解決。
- **開発ネタ**: 大規模IFCファイルや膨大な建築仕様書を扱うAIツールの精度向上に直結。Revitモデルの大規模データを分析するAIアドインで長文脈処理の信頼性を上げる実装手法として使える。

### ★★★ [TestEvo-Bench: An Executable and Live Benchmark for Test and Code Co-Evolution](http://arxiv.org/abs/2607.02469v1)
- ソース: arXiv / カテゴリ: DevTools
- 要約: コード変更に追従するテストの自動生成・更新を評価するベンチマーク。テストとコードの共進化を実行可能なベンチマークで評価する初の試み。
- **開発ネタ**: RevitアドインやIFC処理ライブラリの開発で、コード変更時にテストを自動更新するAIエージェントツール開発のネタ。建築ITの継続的デプロイメント品質向上に活用できる。

### ★★★ [When Do LLM Personas Support Visualization Design?](http://arxiv.org/abs/2607.02455v1)
- ソース: arXiv / カテゴリ: AI
- 要約: LLMペルソナを用いた可視化設計支援において、出力がモデル選択やタスクフレームに依存するアーティファクトかどうかを複数モデルで検証した研究。
- **開発ネタ**: 建築データや設計情報の可視化ツール開発で、LLMペルソナ（設計者・施工者・施主）を使って異なるユーザー視点からUIを評価する手法を取り入れられる。Revitダッシュボードのユーザーテスト代替として使えるかもしれない。
